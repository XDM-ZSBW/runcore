What is {{INSTANCE_NAME}}

  • File-based personal operating system for an AI agent
  • No database — the brain is markdown, YAML, and JSONL files
  • Local-first: runs on the owner's machine, all data stays local
  • Context engineering approach: progressive disclosure of the right information at the right time
  • Optional TypeScript runtime adds programmatic context assembly and file-backed memory

Runtime Stack

  • Dual LLM providers: OpenRouter (cloud) / Ollama (local)
  • Sidecar processes for search, text-to-speech, speech-to-text, and avatar
  • Agent system for autonomous task spawning and batch execution
  • File-backed long-term memory (JSONL append-only stores)

How {{INSTANCE_NAME}} Thinks

Progressive Disclosure

  • Level 1: SKILL.md routes the task to the right module(s)
  • Level 2: Module instruction file loaded (e.g. OPERATIONS.md)
  • Level 3: Specific data files loaded only when needed

Three-Pass Workflow

  • Pass 1 — Intent: Plain language description of what and why
  • Pass 2 — Spec + Build: Technical specification and implementation
  • Pass 3 — Review: Technical review, edge cases, refinement

Memory Model

  • Append-only JSONL stores: experiences, decisions, failures, semantic, procedural
  • Entries carry timestamps, tags, and emotional weight (1–10)
  • Deprecation via "status": "archived" — never delete or rewrite

Protocols

Surface Protein (X-ThoughtStream Header)

Lightweight coordination between distributed nodes. Not authentication — identification. Any node can emit the header; nodes that don't understand it ignore it silently.

FieldTypeDescription
originstringSelf-identification of the emitting node
intentenumThought type: pulse, branch, scar, seed, breath
trace_idstringUnique fingerprint for downstream correlation
backrefstring?trace_id of a related previous thought
weightnumber?0.0–1.0 significance (default 0.5)

Multi-Node Sync

  • JSONL union + dedup across devices (PC primary, phone edge)
  • Transport-agnostic: file sync, git remote, LAN, or personal API
  • Bearing register for conflict resolution, tiered sync for bandwidth
  • Activity entries carry trace_id for lineage tracking
